NCBO BioPortal Ontology Visualization Version History:

Version: 2.3.5
Date: November 9th, 2010
Changes:  
  • When trying to load a deprecated version of an ontology, it now prompts the user if they want to load a different version.
  • Improved the error messages when the server can't load an ontology or concept
  • Fixed a problem where some arc arrows are pointed the wrong way
  • When showing multiple roots, the children aren't loaded anymore (takes too long). If there is only one root then the children are loaded.
  • Added a new layout - Circle Layout Center - which positions the selected nodes in the center of the circle
  • Added a stop button in the progress dialog that lets the user cancel the current operation
  • Added warning dialog when loading the hierarchy to root and more than 50 terms have been loaded so far
  • Improved the tree layouts by ignoring duplicate relationships between the same source and destination nodes
  • Changes to the NCBO search service to handle specical characters like backslash
  • Added a recent ontologies sub-menu to the Ontologies menu
  • Added two new layouts - the ForceDirectLayout and IndentedTreeLayout
  • Many minor fixes and improvements
  • Updated the rest services for the new BioPortal release (v2.2.2). There are now three node types - Class, Individual, and Property.
  • Added new menu items to the Ontology menu for downloading the current ontology and for viewing a different version of the current ontology
  • Added autocompletion using the NCBO Search service in the Ontology Tree search box
  • Changed all occurrances of "Concept" to "Term"
  • When the ontology tree and the flexviz graph are linked the selection is now copied from the graph to the tree
  • Added support for setting the virtual="true" parameter so that the virtual ontology id can be used instead of the version id
  • Some changes to the graph and how node double click events are handled
  • Added expand/collapse double click support in the basicflexviz
  • Added more parameters for better configuration when used as a widget
  • Added a new functions to IExternalGraphInterface for more JavaScript support
  • Some changes to the rest services and BioPortal urls
  • Added ontology groups support to the OntologyChooser
  • Fixed arc rendering bug when lots of arcs were displayed (arcs wouldn't show up)
  • Improved the support for custom node and arc renderers and styles
  • Fixed up the JavaScript API
  • Added export to XML/Image option (in main toolbar)
  • Fixed a bug that happens when searching from the FlexViz tree
  • Changed some of the context menus on the graph
  • The Ontology Tree expansions and collapses are now linked with the graph
  • Added a close button to the ontology tree as well as a hide/show menu item in the Ontology menu
  • Update to incorporate changes to the underlying graph component (grouped nodes, marquee node selection)
  • Improved filtering
  • Minor fix to the ontology tree where a null pointer was happening on startup
  • Improved the "Link To This Concept" context menu item - now a popup window is shown with options for copying urls to FlexViz, BioPortal, or copying the iframe embed html code.
  • Minor fixes to the ontology tree synchronization
  • The Spring layout when used on a node being expanded now runs a fisheye algorithm first to make space for the children
  • Released with BioPortal 2.1
  • Added the ontology tree on the left of FlexViz
  • Added up to 3 properties panes in the right panel for showing the selected node properties
  • Added a "Link To Concept" context menu item for bookmarking a concept
  • A minor fix in the rest services and error handling
  • Added history navigation (back/forwards)
  • Removed some icons
  • Removed the root nodes (owl:Thing/THING)
  • Improved the search autocomplete box
  • Added the Spring layout algorithm
  • Improved the "Fit to screen" and "Expand graph" actions
  • Fixed a bug where nodes and arcs are shown even when the node/arc type is filtered
  • Changed the Search bar to prompt the user whether he/she wants to continue searching even when a node has the exact text that was entered in the search. If the user chooses yes then a search call is made to the REST services.
  • Fixed the xml parser to work with the latest REST services
  • Added an ontology category filter combobox to the Ontology Chooser window, and a label which shows the selected ontology
  • Added three more application parameters: adminemail (the admin email address), bugurl (the bug tracking url), and title (browser title)
  • Added a "Properties" option to the nodes context menu which pops up a resizable window showing all the properties of the concept
  • Added a warning dialog which gets displayed before a node is expanded IF it has more than 50 children
  • Fixed the ontology chooser dialog to only display local ontologies (remote ones can't be visualized)
  • Some minor performance improvements
  • Added a menu item under the Nodes menu which lets you choose which property to use for node labeling (e.g. Prefered_Name, rdfs:label, etc)
  • Added shortcut keys for showing/focussing on the neighborhood (N) and hierarchy to root (H) of the selected node
  • Changed the graph to now have a blue border when it is focussed
  • Changed the plus icon to be rendered below the node indicating the it has hidden children
  • Added a new search results panel for displaying multiple search results (instead of just showing the neighborhood for all)
  • Added "part of" to the list of inverted arc types
  • Also added an "invert" menu item arcs context menu in the full version, it changes the arrow head direction
  • Fixed the tree and grid layouts to ensure that a minimum separation of 20 pixels exists between each node
  • Changed the graph to center on a focussed node after the layout animation has finished, it is smoother this way
  • Added a warning dialog when too many (>40 nodes) are going to be displayed on the screen, this warning can be turned off
  • Improved the node and arc tooltip configuration (under the Node and Arc menus)
  • Added support for passing in the ontology and nodeid parameters in the browser URL. By default it looks at the application parameters first, then the browser url.
  • Fixed some problems with the REST XML parser and adding relationships
  • Added a double click handler on the BasicFlexoViz which opens the direct link to the clicked concept in the current window
  • Minor fixes to the vertical tree layout when there are multiple roots in the tree
  • Added scroll-to-node support so that a single node can be centered in the screen when the graph has scrollbars
  • Added all concept properties as a tooltip, including comments, definitions, synonyms, etc


Version: 2.0.0
Date: August 8th, 2008
Changes:
  • Released with BioPortal 2.0
  • Added the 'Hide Orphans' action under the Arc Types panel - this action hides all unconnected nodes
  • Improved debugging support
  • Added some content to the help panel on the ontology navigation actions
  • Added the "Fit To Screen" and "Expand Graph" actions to the full FlexoViz toolbar
  • In the full version you can now choose which ontology to load from the Ontology menu
  • Added descriptions, comments, and synonyms to concept tooltips
