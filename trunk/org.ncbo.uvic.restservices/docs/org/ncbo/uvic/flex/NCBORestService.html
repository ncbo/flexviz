Packageorg.ncbo.uvic.flex
Classpublic class NCBORestService
ImplementsIRestService

Main class for working with the NCBO Rest services. Handles caching of ontologies and their concepts. It also caches calls to the rest service to prevent identical calls from happening, instead the callbacks functions are saved and will get called when the response comes back. It can also retry a rest service call if it fails.

getConceptByID()method 
public function getConceptByID(ontologyID:String, conceptID:String, callback:Function, loadNeighbors:Boolean = false, alertErrors:Boolean = true):void

Gets a concept from an ontology by ID. The callback function should accept one parameter - an NCBONodeEvent object. This function will first try to find the node from the cache, if it can't be found then it will make a call to the web service to find the node.

Parameters
ontologyID:String — the id of the ontology
 
conceptID:String — the id of the concept to find
 
callback:Function — the function to call when the concept has been found (takes one param - NCBOConceptEvent)
 
loadNeighbors:Boolean (default = false) — if true and if the concept doesn't have its neighbors loaded yet we make a call to the rest services to load concept and therefore the neighbors too.
 
alertErrors:Boolean (default = true) — if true and and error happens, an Alert box will show up displaying the error

public function getConceptsByName(ontology:NCBOOntology, conceptName:String, mode:SearchMode, callback:Function):void

Searches for concepts by name in a single ontology. Currently the mode only supports contains (default) or exact match. The callback will take an NCBOSearchEvent parameter containing the matching ontologies (which there should only be one) and the matching concepts for that ontology. Note that the search results will contain NCBOSearchResultConcept objects, not the full NCBOConcept objects.
