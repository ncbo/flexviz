Interfacepublic interface IRestService
ImplementorsNCBORestService

Defines the following functions of the NCBO Rest Service:
  • get all ontologies
  • get an ontology by id
  • get the roots of an ontology
  • get a concept by id
  • search for all concepts by name
  • get the parents of a concept
  • get the children of a concept

    getConceptByID()method 
    public function getConceptByID(ontologyID:String, conceptID:String, callback:Function, loadNeighbors:Boolean = false, alertErrors:Boolean = true):void

    Gets a concept from an ontology by ID. The callback function should accept one parameter - an NCBONodeEvent object. This function will first try to find the node from the cache, if it can't be found then it will make a call to the web service to find the node.

    Parameters
    ontologyID:String — the id of the ontology
     
    conceptID:String — the id of the concept to find
     
    callback:Function — the function to call when the concept has been found (takes one param - NCBOConceptEvent)
     
    loadNeighbors:Boolean (default = false) — if true and if the concept doesn't have its neighbors loaded yet we make a call to the rest services to load concept and therefore the neighbors too.
     
    alertErrors:Boolean (default = true) — if true and and error happens, an Alert box will show up displaying the error
    getConceptsByName()method 
    public function getConceptsByName(ontology:NCBOOntology, conceptName:String, mode:SearchMode, callback:Function):void

    Searches for concepts by name. This function only works on one ontology, which has to be passed in as a paramter. To search multiple ontologies, use the search() function instead. Currently the mode only supports contains (default) or exact match. The callback will take an NCBOSearchEvent parameter containing the search results.

    getTopLevelNodes()method 
    public function getTopLevelNodes(ontologyID:String, callback:Function):void

    Loads the root nodes for the given ontology. If the ontology has already loaded the roots, then the callback function is called immediately. Otherwise the rest service will go get the roots and then call the callback. The root nodes that get returned are not "owl:Thing" or "THING", instead they are the children of "owl:Thing" or "THING".

    Parameters
    ontologyID:String — the function which gets called with the NCBOConceptsEvent object
     
    callback:Function
    search()method 
    public function search(searchParams:SearchParams, callback:Function, parseHits:Boolean = true, logEvent:Boolean = true):void

    Searches across multiple ontologies. Also supports paging, set the flags in the SearchParams object to get a specific page. Currently the mode only supports contains (default) or exact match. The callback will take an NCBOSearchEvent parameter containing the search results. Note that the search results will be NCBOSearchResultOntology objects containing NCBOSearchResultConcept objects. These objects will have the same id's as the NCBOOntology and NCBOConcept objects, but not as much other information.

    Parameters
    searchParams:SearchParams — the search parameters, includes the search text, the mode (contains/exact match) and the ontology ids.
     
    callback:Function — if false then the search event won't be logged
     
    parseHits:Boolean (default = true)
     
    logEvent:Boolean (default = true)
